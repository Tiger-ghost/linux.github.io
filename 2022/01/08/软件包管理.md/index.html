软件包管理

包命名

xorg-x11-xkb-utils-7.7-12.el7.x86_64.rpm
xorg-x11-xkb-utils:软件名
7.7:版本
12:是编译次数
el7:适合什么版本，这是centos7上的rpm包就显示el7 centos6就会显示el6
x86_64:表示适合的硬件平台

统计Package下所有的软件包，有多少种硬件平台

库文件

复制源码查看二进制程序所依赖的库文件
ldd /PATH/TO/BINARY_FILE
管理及查看本机装载的库文件
ldconfig 加载配置文件中指定的库文件
/sbin/ldconfig -p 显示本机已经缓存的所有可用库文件名及文件路径映射关系
配置文件:/etc/ld.so.conf,/etc/ld.so.conf.d/*.conf
缓存文件:/etc/ld.so.cache

删除tree命令并找回

tree命令包含很多文件，这里只是删除了一个文件： /usr/bin/tree，
所以方法一：可以覆盖安装。使用--replacepkgs
方法二：只恢复删除的文件:
rpm2cpio 针对某个rpm包解包或者预览

--nosignature:不检查来源合法性

安装软件会有一个报警提示warning加了这个参数就没有了

卸载rpm

Centos7卸载rpm命令后，进救援模式重新安装rpm
rpm -ivh XX --root=/mnt/sysimage
进入镜像修复安装，注意安装的路径，后面加上--root=/mnt/sysimage才是安装到系统。

包校验

每个点表示一个类型：
S 文件大小是否改变
M 文件的类型或文件的权限（rwx）是否变改变
5 文件MD5校验和是否改变（可以看作文件内容是否改变）
D 装置的主/次代码已经改变
L Link路径已被改变
U 文件的所属人已被改变
G 文件的所属群组已被改变
T 文件的创建时间忆被改变
rpm -Va 校验所有的rpm包

rpm包公钥文件:就在挂载光盘目录下的 RPM-GPG-KEY-CentOS-7
将它导入:rpm --import RPM-GPG-KEY-CentOS-7这样以后安装就不会有报警了。

也可以不从光盘里面导入，只要装完系统cd /etc/pki/rpm-gpg/此路径下就有一个公钥文件，和光盘里面的是一样的。直接导入也可以。

自建yum仓库

如果是自己下载的RPM包，没有仓库的元数据，是无法使用yum,所以需要在当前目录下创建元数据。
createrepo .在rpm包文件夹下直接执行，就可生成元数据。
createrepo命令会分析每个RPM包的元数据并提取出来，放在repodata(自动新建)目录下。

程序包编译

程序包编译安装:
源代码——预处理——编译——汇编——链接——执行
源代码组织格式:
   多文件:文件中的代码之间，很可能存在跨文件依赖关系
   C、C++:make 项目管理器
   configure脚本 –> Makfile.in –> Makefile
   java:maven

编译安装tree

rpm -qi tree 查询软件包信息得到URL，再到链接网站上 复制下载链接
wget http://mama.indstate.edu/users/ice/tree/ 回到系统上直接下载
tar xvf tree-2.0.1.tgz 解压缩，得到解压后的文件夹
cat INSTALL 查看安装步骤
PREFIX=/apps/tree Makefile文件中定义安装路径
make时，Centos7.9安装会有一个报错：需要修改Makefile文件中的:CC=/usr/bin/c99
make INSTALL 后就可以用新的tree了，但是系统也有一个tree,所以改变环境变量就可以执行了

编译安装httpd-2.4

./configure
–prefix=/apps/httpd24
–sysconfdir=/etc/httpd
–enable-ssl
–enable-so
缺包就是后面加devel,执行完成以后就会生成makefile文件，
make -j 4(看有几个CPU)
make install
cd /apps/httpd24/bin/ 查看文件已经有了

浏览器打来验证。但是这样只是现在打开了，无法自启动。将它加入一个文件

页面来源文件路径：/apps/httpd24/htdocs/index.html
