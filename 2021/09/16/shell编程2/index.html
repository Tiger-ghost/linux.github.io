使用read来把输入值分配给一个或多个shell变量
-p 指定要显示的提示
-s 静默输入(不在屏幕上显示)，一般用于密码
-n N 指定输入的字符长度N
-d ‘字符’ 输入结束符
-t N TIMEOUT为N秒
read 从标准输入中读取值，给每个单词分配一个变量，所有剩余单词都被分配给最后最后一个变量

shell登录两种方式

交互式登录:
(1)直接通过终端输入账号密码登录
(2)使用”su - UserName” 切换的用户
执行顺序:/etc/profile --> /etc/prifile.d/*.sh --> ~/.bash_profile --> ~/.bashrc --> /etc/bashrc
非交互式登录:
(1)su Username
(2)图形界面打开终端
(3)执行脚本
(4)任何其他的bash实例
执行顺序:/etc/profile.d/*.sh --> /etc/bashrc --> ~/.bashrc

bashrc类:为非交互式和交互式登录的shell提供配置

Bash 退出任务

保存在~/.bash_logout文件中(用户)
在退出登录shell运行
用于
-创建自动备份
-清除临时文件
这个文件就是当用户退出时，自动运行此文件里面的命令

set命令

$- 变量
h:hashall，打开这个选项后，shell会外部命令的路径hash下来,避免每次都要查询。通过set +h将h选项关闭。查看缓存命令:hash
i:interactive-comments，包含这个选项说明当前的shell是一个交互式的shell。所谓的交互式shell,在脚本中，i选项是关闭的。
m:monitor，打开监控模式，就可以通过Job control来控制进程的停止、继续，后台或者前台执行等。
B:braceexpand,大括号扩展
H:history,H选项打开，可以展开历史列表中的命令，可以通过!感叹号来完成，例如!! 返回上最近的一个历史命令，!n返回第n个历史命令。

脚本安全

set命令:
-u: 当使用一个没有定义的变量的时候，显示错误信息， 等同set -o nounset
-e: 如果一个命令返回一个非0退出状态值(失败)就退出等同set -o errexit

linux中()中的命令不是在子shell中执行，但是为什么可以使用父shell中的变量？
