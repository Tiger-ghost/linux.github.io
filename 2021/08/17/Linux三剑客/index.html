linux-正则表达式

Posted on 2021-08-17 Edited on 2021-08-23 In 笔记
grep+正则表达式

grep

文本过滤(模式:pattern)工具.打印匹配模式的行
作用:文本搜索工具,根据用户指定的”模式”对目标文本逐步进行匹配检查；打印匹配到的行
模式:由正则表达式字符及文本字符所编写的过滤条件

--color=auto

对匹配到的文本着色显示，有些用户有定义别名alias默认显示这个;

-m

只匹配m次后停止
grep的原理是读取文件中的一行，到自己的内存空间中分析，如果有匹配的行就打印到屏幕上，然后把读取到的清理到，再读取下一行。

-v

显示不包括的行，grep -v root /etc/passwd表示显示不包括root的行;

-i

忽略字符大小写

-n

显示匹配到的字符的行号

-c

显示匹配到的字符有多少行

-o

只显示匹配到的字符串，可以看到有多少个。

-q

静默显示，不输出任何信息

-A #

显示后#行

-B #

显示前#行

-C #

前后各#行

-e

实现多个选项间的逻辑or关系

-w

匹配整个单词，数字 字母 下划线才算单词,其他都算分隔符;

-F

相当于fgrep,不使用正则表达式
grep -F root /etc/passwd 相当于搜索的是一个普通的字符串，不是正则表达式

-f

根据模式文件处理

-E

使用ERE扩展正则表达式

正则表达式

REGEXP:Regular Expressions,由一类特殊字符及文本字符所编写的模式，其中有些字符(元字符)不表示字符字面意义，而表示控制或通配的功能。
程序支持:grep sed awk vim less nginx varnish等
分两类:
基本正则表达式: BRE
扩展正则表达式: ERE
grep -E,egrep
正则表达式引擎:
采用不同算法,检查处理正则表达式的软件模块。
PCER(Perl Compatible Regular Expressions)
元字符分类:字符匹配、匹配次数、位置锚定、分组
man 7 regex

匹配次数

用在要指定次数的字符后面，用于指定前面的字符要出现的的次数

位置锚定

位置锚定：定位出现的位置

分组

\(\)将一个或多个字符捆绑在一起，当作一个整体处理，如:\(root\)\+
分组括号中的模式匹配到的内容会被正则表达式引擎记录与内部的变量中，这些变量的命令方式为:\1,\2,\3,……
\1表示从左侧起第一个左括号以及与之匹配右括号之间的模式所匹配到的字符：

后项引用:引用前面的分组括号中的模式所匹配字符，而非模式本身。或者：\|

egrep及扩展的正则表达式

扩展正则表达式和之前最大的不同就是不用加斜线除以下需要加：
grep -E 等价于 egrep

示例：

grep -v "^#" /etc/fstabs 和 grep "^[^#]" /etc/fstabs 的差别
