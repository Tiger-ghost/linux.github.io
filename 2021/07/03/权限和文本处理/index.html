权限-文本

文件权限

文件属性

chgrp

修改文件属组，只能修改文件的属组。
-R:递归；

chown

修改文件的属主和属组。

chmod

–reference:参照其他文件的权限来设置新文件。

权限数字法：

目录的权限用途：

读：可以列出文件名；
执行：可以进入目录，可以访问目录的文件内容；
写：创建和删除文件；

mkdir

mkdir /data -m 700:创建文件夹时就设置权限；

linux文件系统上的特殊权限

SUID,SGID,Sticky
三种常用权限：r,w,x, user,group,other
安全上下文

提问

cp /etc/fstab /data/dir/ 普通用户需要什么最少权限？
cp 需要用执行权限
/etc/fstab /etc/至少有执行权限，fstab需要有读权限；
/data/dir/ /data至少有执行权限，/data/dir/至少有xw权限；

chattr设定文件特殊属性

umask

umask+default=777目录 ｜ 666文件
umask功能：将777｜666中对应的权限去掉，得出默认权限

默认权限不能直接进行减法处理，因为文件不能直接拥有x权限，必须：
文件：666-umask,如果有奇数就+1，偶数不变；
目录：777-umask；

ACl访问控制列表

ACL：access Control list，实现灵活的权限管理。
除了文件的所有者，所属组和其他人，可以对更多的用户设置权限
Centos7默认创建的xfs和ext4 文件系统具有ACL功能
Centos7之前的版本，默认手动创建的ext4文件系统无ACL 功能，需手动增加

ACL生效顺序：所有者，自定义用户，自定义组，其他人

文本处理

cat

将文件保存为windows文件：

tac: 和cat相反，倒着显示内容。

nl:相当于cat -b filename 对非空行加编号。

rev

分页查看文件内容

more:分页查看文件,-d显示翻页及退出提示
less:分页查看文件，查看时可搜索：/文本 搜索
n/N 跳到下一个或上一个匹配
空格向下翻页，回车向下以行递增，b向上翻页。
less 命令是man命令使用的分页器。

head

默认显示文本前10行
-c # 指定获取前#个字节
-n # 指定获取前#行
-# 指定行数(可以不加-n)

tail

默认显示文本倒数10行
-c # 指定获取后#字节
-n # 指定获取后#字节
-# 同上
-f:跟踪显示文件fd新追加的内容，常用日志监控相当于–follow=descriptor
这里跟踪的是文件的描述符，如果文件被删除了就检测不到了，即便新建一个名字一样的文件也是不行的。

-F: 跟踪文件名，相当于–follow=name –retry
和-f相反，这里他只看文件名，即使文件被删除了，只要重新建立一个与之前文件同名的文件那么他就会继续开始跟踪。

tailf

类似tail -f,当文件不增长时并不访问文件。

cut

按列抽取文本

-c 选取字符的列表，即选取每个的第N个字符。
(英文字符下与-b没有区别，中文字符下，一个中文占据2-3个字节，所以存在中文的时候更倾向于用-c)。

–output-delimiter=STRING指定输出分隔符；

paste

cat f1.txt f.txt 输出结果为纵向合并；上下

paste为横向合并 左右
-d 分隔符：指定分隔符，默认用TAB
-s ：所有行合并成一行显示。

sort

文本排序默认使用ASCII进行排序。
把整理过的文本显示在stdout，不改变原始文件sort[options]files
-r:执行反方向排序，降序；
-R:随机排序；
-n:执行按数字大小整理；
-f:选项忽略(fold)字符串中的字符大小写;
-u:选项(独特,unique)删除输出中的重复行；
-t :指定分隔符；
-k :指定列数；

uniq

从输入中删除前后相接的重复的行
-c:显示每行重复出现的次数
-d:仅显示重复过的行
-u:仅显示不曾重复的行
注:连续且完全相同方为重复；
常和sort命令一起配合使用：
sort file.txt | uniq -c

diff

这个时候我可以把这个输出存为一个文件，diff.log 如果b.txt 被删除了，我可以用
通过a.txt来恢复b.txt文件，使用patch命令即可。但是无法通过b.txt来恢复a.txt

wc

统计指定文件中的字节数、字数、行数，并将统计结果显示输出。该命令统计指定文件中的字节数、字数、行数。如果没有给出文件名，则从标准输入读取。wc同时也给出所指定文件的总统计数。
-c 统计字节数。
-l 统计行数。
-w 单词数
