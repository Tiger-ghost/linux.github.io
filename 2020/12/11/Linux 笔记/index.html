目录简介

/proc /sys 比较特殊的目录，他们目录的数据大小为0
/proc 放的是和进程相关的 里面的数据是内存中的，不在磁盘上
/sys 里面的是硬件信息

标准的linux文件系统（如ext4），文件名称大小写敏感
fat32的文件系统大小写不敏感，就像windows里面的一个分区 挂在到 linux 中，在linux中就把这个分区当成一个磁盘，这个磁盘的文件系统是fat32的，里面可以不用区分大小写。

创建带-的文件：touch -- -a.txt
删除：rm -- -a.txt

pwd
-P 显示真实物理路径
-L 显示链接路径（默认）

basename 取文件名(基名)
dirname 取路径名

文件时间

查看一个文件的最后的时间是指文件的最后的修改时间
ll a.txt
-rw-r–r–. 1 root root 0 Dec 3 14:07 a.txt
Dec 3 14:07 这个时间是指文件的最后修改时间

查看SElinux 状态：getenforce

Linux下文件的三个时间参数：stat a.txt
1）modification time(mtime):内容修改时间
这里的修改时间指的是文件的内容发生变化，而更新的时间。

2）change time(ctime):状态修改时间
这里的修改时间指的是文件的属性或者权限发生变化，而更新的时间。

3）access time(atime):最后访问时间
这里的访问时间是指文件被读取，而更新的时间。

文件管理基本命令cp和mv

cp命令能复制文件，但是文件上的有些信息会丢失；也可以一次性复制多个文件，但要求目标是一个目录(而且是已经存在的目录)。
复制文件夹需要加选项：-r

例：cp -r /etc/sysconfig /data/a
如果/data/下面没有a文件夹，那么/sysconfig文件夹会复制过来并且文件夹改名为a；如果a文件夹和已存在那么/sysconfig目录会复制到a的文件夹下面。

用原始命令不用别名就在命令的前面加\
在root下 alias cp=’cp -i’ -i表示会提示复制是否覆盖原文件，想要不提醒就在命令的前面加\

-d 如果源文件为软链接（对硬链接无效），则复制出的目标文件也为软链接；

如果想要保存所有的信息就用-a，一般来充当备份的效果
-a 相当于-dR --preserv=all

如果想要看保存的过程加选项 -v

复制备份：cp --backup=numbered /etc/issue f1 这个选项表示，无论复制多少次，目标文件如果存在都会加数字备份。

文件节点表和原理

索引节点

inode(index node)表中包含文件系统所有文件列表
一个节点(索引节点)是在一个表项，包含有关文件的信息(元数据)，包括：
文件类型，权限，UID，GID
链接数(指向这个文件名路径名称个数)
该文件的大小和不同的时间戳
指向磁盘上文件的数据块指针
有关文件的其他数据

关于文件夹来讲，数据栏那一块放的是文件夹下文件的文件名

文件链接类型

硬连接

硬连接的本质就是一个文件创建多个名字。
硬链接不支持对文件夹进行创建会造成死循环嵌套的问题。也不支持跨分区创建，因为本质上硬链接是同一个文件 夸了设备分区就不是同一个文件了。

-rw-r--r-- 2 root root 0 Dec 11 14:29 a1
这里的2表示 链接数，自己本身就算一个，还有一个硬链接

而一个新建的文件夹的链接数默认是有两个，自己本身算一个，文件夹里面的隐藏文件.也算一个。以此类推 如果文件夹里面继续新建文件其他的文件夹，那么还有..也算他的链接数。所以说不管新建的文件夹下面的文件夹有多少个， 只要能表示这个新建的文件夹的，链接数就会增加。可以 ll -i 查看，只要inode号一样就表示同一个文件，链接数就会增加。

软链接

软链接就是不同文件，软链接支持对目录的创建
创建软链接时，要用相对路径时，他的原始文件相对路径一定是相对软链接的路径。不是相对于当前目录的路径。

删除文件夹的软链接时请注意：
如果一个文件夹f f的下面还有其他文件a b
ln -s f f.link 创建了f的软链接f.link
rm -fr f.link/ 删除文件夹的软链接时后面不可以带/，这样删除时会将文件夹里面的文件全部删除，并不是删除软链接本身。

用户和组的配置文件

Linux用户和组的主要配置文件：
/etc/passwd：用户及其属性信息(名称，UID,主组ID等)getent passwd 用户名(可选)
/etc/group：组及其属性信息.getent group 可以直接进行查看
/etc/shadow：用户密码及其相关属性
/etc/gshadow：组密码及其相关属性

用户和组管理命令

用户管理命令：

useradd
-u: UID
-o: 配合-u选项，不检查UID的唯一性(为了和某些账号的权限一样，将UID设置为一样。用户名只是作区分)
-g: GID 指明用户所属基本组，可为组名，也可GID
-Ggroup1[,group2,..]: 为用户指明附加组，组必须事先存在
-N: 不创建私用组做做主组，使用users组做主组

-r: 创建系统用户Centos6:ID<500,Centos7:ID<1000(-r默认不会创建家目录)
-m: 创建家目录，用于系统用户。
-M: 不创建家目录，用于非系统用户。(因为创建系统用户默认会自动创建家目录)
-sSHEll: 指明用户默认的shell程序，可用列表在/etc/shells 文件中
-dHOME_DIR: 以指定的路径(不存在)为家目录

-c“COMMENT”: 用户的注释信息 (使用finger可以进行查看)
useradd的默认属性 在文件：cat /etc/default/useradd,这个文件里面有一个 SKEL=/etc/skel 这是一个模板目录，就是说用户账号家目录创建的时候会自动复制这个目录中的文件到新建的家目录中。
useradd -D 也能直接看到useradd的属性文件。

usermod
-u UID: 新UID
-g GID: 新主组
-G GROUP1,[GROUP2],..: 新附加组，原来的附加组会被覆盖，如果要保留原有，则要同时使用-a选项
-s SHELL: 新的默认SHELL
-c 'COMMENT': 新的注释信息。
-d HOME: 新家目录不会自动创建；若要创建新家目录并移动原家目录数据，同时使用-m选项。
-l login_name: 新的名字
-L: lock指定用户，在/etc/shadow 密码栏的增加！
-U: unlock指定用户，在/etc/shadow密码栏的！拿掉
-e YYYY-MM-DD: 指明用户账号过期日期
-f INACTIVE: 设定非活动期限。

userdel
-r: 删除家目录

ssh 基于key的认证

基于密钥的认证：
1.首先在客户端生成一对密钥(ssh-keygen)
2.并将在客户端的公钥ssh-copy-id 拷贝到服务端
3.当客户端再次发送一个连接请求，包括ip，用户名
4.服务端得到客户端的请求后，会到authorized_keys中去查找，如果有响应的ip和用户，就会随机生成一个字符串，例如magedu
5.服务端将使用客户端拷贝过来的公钥进行加密，然后发送给客户端
6.得到服务端发送过来的消息后，客户端会使用私钥就行解密，然后将解密后的字符串发送给服务端。
7.服务端接受到客户端发来的字符串后，跟之前的字符串进行对比，如果一致，就允许免密码登录。

当用户远程链接ssh服务器时，会复制ssh服务器/etc/ssh/ssh_host*key.pub（centos7默认是ssh_host_ecdsa_key.pub）文件中的公钥到客户机的~./ssh/know_hosts中。(也就是说knows_hosts这个文件就是曾经连过的对方主机的公钥)下次链接时，会自动匹配到相应的私钥，不能匹配，将拒绝链接。
